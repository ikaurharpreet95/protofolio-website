# Airtable Integration Setup Guide

This guide will help you connect your Airtable base to the contact form on your portfolio website.

## Prerequisites

- An Airtable account (free or paid)
- An existing Airtable base
- Node.js and npm installed

## Step 1: Create/Prepare Your Airtable Table

1. Go to [Airtable](https://airtable.com) and sign in
2. Open or create a base
3. Create a table named **"website"** (or your preferred name)
4. Add the following fields to your table:
   - **Name** (Single line text)
   - **Email** (Email)
   - **Phone** (Phone number)
   - **Event Type** (Single select or Long text)
   - **Message** (Long text)
   - **Submission Date** (Date/time - auto-generated by the form)

### Table Structure Example:
```
Field Name          | Field Type
--------------------|------------------
Name               | Single line text
Email              | Email
Phone              | Phone number
Event Type         | Single select
Message            | Long text
Submission Date    | Date (created time)
```

## Step 2: Get Your Airtable API Key

1. Go to [Airtable Developer Hub](https://airtable.com/create/tokens)
2. Click **"Create new token"**
3. Give your token a name (e.g., "Portfolio Website")
4. Under **"Scopes"**, select these permissions:
   - `data.records:write` - Write form data
   - `data.records:read` - Read records (optional)
5. Click **"Create token"**
6. Copy the token (you won't see it again!)

## Step 3: Get Your Base ID

1. Open your Airtable base in your browser
2. Look at the URL: `https://airtable.com/app/[BASE_ID]/...`
3. Copy the `BASE_ID` part

## Step 4: Configure Environment Variables

1. Open `.env.local` in your project root
2. Add your credentials:
   ```
   VITE_AIRTABLE_API_KEY=your_personal_access_token_here
   VITE_AIRTABLE_BASE_ID=your_base_id_here
   VITE_AIRTABLE_TABLE_NAME=website
   ```

3. **Important**: Never commit `.env.local` to git. It's already in `.gitignore`.

## Step 5: Test the Integration

1. Start your development server:
   ```bash
   npm run dev
   ```

2. Open the website in your browser (http://localhost:3000)

3. Fill out the contact form and submit

4. Check your Airtable table - you should see a new record with the form data!

## Troubleshooting

### "Missing Airtable configuration" Error
- Make sure `.env.local` exists and has all three variables set
- Restart the dev server after updating `.env.local`
- Check for typos in environment variable names

### "Failed to submit form" Error
- Verify your API key is correct
- Check that the Base ID is accurate
- Ensure the table name matches exactly (case-sensitive)
- Confirm your token has `data.records:write` permission

### Records not appearing in Airtable
- Check browser console for error messages
- Verify field names in Airtable match the expected format
- Check that your Airtable token hasn't expired

### CORS Errors
- This shouldn't happen with Airtable's official API
- Check that you're using the latest version of the code

| Form Field | Airtable Field | Type |
|-----------|----------------|------|
| Name | Name | Text |
| Email | Email | Email |
| Phone | Phone | Phone |
| Event Type | Event Type | Text/Select |
| Message | Message | Long Text |
| Submission Time | Submission Date | DateTime |

## Security Notes

- **Never** share your API key
- **Never** commit `.env.local` to version control
- Rotate your token if you suspect it's been compromised
- The token should only have write access to the specific table

## Production Deployment

When deploying to production:

1. For **Vercel**: Add environment variables in project settings
2. For **Netlify**: Add environment variables in Build & deploy settings
3. For **Other hosts**: Follow your host's environment variable documentation

Set the same three variables:
- `VITE_AIRTABLE_API_KEY`
- `VITE_AIRTABLE_BASE_ID`
- `VITE_AIRTABLE_TABLE_NAME`

## Need Help?

If you encounter issues:
1. Check the browser console for error messages
2. Verify all credentials in `.env.local`
3. Ensure your Airtable token hasn't expired
4. Check the Airtable API status page
